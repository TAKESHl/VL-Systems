As telas representam como o aplicativo irá funcionar para o cliente, pois para os demais atores irão funcionar de forma um pouco diferente.

No diagrama de casos de uso, nós temos 4 atores, sendo eles: O Cliente, o atendente, o Pizzaiolo e o Motoboy. No modelo, as interações se iniciam no cliente, onde ele realiza o pedido através do aplicativo, nele, o cliente poderá escolher o sabor das pizzas, se deseja refrigerante e por fim adicionar o endereço de entrega (Como demonstrado em cada uma das telas incluídas neste trabalho). Após isso, irá definir a forma de pagamento e realizá-lo imediatamente ou não (se for escolhido pagamento em dinheiro). O Atendente irá receber esse pedido e passar todos os detalhes para o pizzaiolo. E por fim, na interação final, o motoboy verifica e retira os pedidos que são necessários realizar a entrega (Este possui acesso ao sistema da pizzaria também para verificação). No momento da entrega, o cliente realiza o pagamento das pizzas se caso tenha escolhido pagar no dinheiro. Este diagrama representa um Sistema distribuído pois para o funcionamento deste requer comunicação entre todos os seus atores, até a entrega do pedido, todos eles conversam entre si para o sistema como um todo funcionar de forma correta.
